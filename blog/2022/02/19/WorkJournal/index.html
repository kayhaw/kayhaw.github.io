从2021年9月份到2022年2月份的工作日志总结

日常🤡​

测试人员发现项目升级后，以前跑的任务运行日志没有数据了，但是现在验证的任务可以看到运行日志。使用Cerebro查询数据后发现ES中确实没有运行日志，可以确认ES迁移没有把数据转过来，或者ES配置只保留最近N天的日志。遂将BUG标记为外部原因，让测试去问运维。接下来简单讲下怎么使用Cerebro来查询ES日志。

下载Cerebro压缩包，解压后双击进入bin目录双击cerebro.bat文件。弹出黑框显示如下内容表示服务启动，然后在浏览器输入http://localhost:9000开始使用。

[info] play.api.Play - Application started (Prod) (no global state)
[info] p.c.s.AkkaHttpServer - Listening for HTTP on /0:0:0:0:0:0:0:0:9000

然后在开着Cerebro服务的情况下尝试在本地起同步引擎服务，但是服务端口号也是9000，导致项目启动失败。

任务运行时间超长​

某项目微服务升级，同步引擎执行节点和同步引擎provider服务都迁移到了另一个ip地址，但是迁移的时候还有任务在运行，任务完成后provider服务回调更新任务状态，但更新的是旧业务库，新业务库一直没有得到更新，因此一直显示运行中导致运行时间超长。

脏数据和脏代码​

任务信息导出为Excel文件，该方法执行时遇到脏数据字段为空抛出了空指针异常，但是代码没有catch异常而是直接在final块中关闭输出流，导致前端下载的Excel文件格式损坏不能打开(看起来文件能下载但就是打不开)。解决方案：在写Excel文件时判断是否有脏数据，有的话直接跳过。

Redis Database(RDB)方式Redis每隔一段时间将内存中数据集中写入磁盘快照，恢复时将快照文件加载到内存。该快照文件名默认为dump.rdb，使用redis-server启动redis服务时，会在当前目录生成dump.rdb文件，在redis.conf中与rdb相关的配置有：

  1. dir：rbd文件目录
  2. dbfilename：rdb文件名 3
